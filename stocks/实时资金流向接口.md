# 实时资金流向接口说明（东方财富，免费无 Key）

> 用途：一次请求拿到**全 A 股每只股票的当日累计资金流向**，按主力净流入排序。
> 等同于东财网页 [data.eastmoney.com/zjlx](https://data.eastmoney.com/zjlx/) 那张全市场资金流排行表的数据源。
> 实盘时盘中调用即为「当前时刻累计」的实时值，收盘后定格为全天终值。

---

## 一、全市场资金流排行接口（一次拿全量）★主用

### 接口地址
```
https://push2.eastmoney.com/api/qt/clist/get
```
分流域名（被限流时可轮换，行为一致）：`push2` / `82.push2` / `64.push2`

### 完整请求示例（curl）
```bash
curl -s -H "Referer: https://quote.eastmoney.com" \
  -H "User-Agent: Mozilla/5.0" \
  "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=5000&pn=1&np=1&fltt=2&invt=2&fs=m:0+t:6,m:0+t:80,m:1+t:2,m:1+t:23&fields=f12,f13,f14,f2,f3,f62,f184,f66,f69,f72,f75,f78,f81,f84,f87"
```

### 请求参数说明
| 参数 | 取值 | 含义 |
|------|------|------|
| `fid` | `f62` | 排序字段。f62=按主力净流入额排序；也可用 f184(按主力净占比) |
| `po` | `1` | 排序方向。1=降序（净流入从高到低），0=升序 |
| `pz` | `5000` | 每页数量。设 5000 一次即可覆盖全 A 股（约 5400 只，必要时翻页） |
| `pn` | `1` | 页码 |
| `np` | `1` | 固定 |
| `fltt` | `2` | 数值格式，2=返回标准小数 |
| `invt` | `2` | 固定 |
| `fs` | 见下 | 市场板块筛选 |
| `fields` | 见下 | 需要返回的字段列表 |

### `fs` 市场筛选取值
| fs 值 | 范围 |
|-------|------|
| `m:0+t:6,m:0+t:80` | 深市 A 股（主板 t:6 + 创业板等 t:80） |
| `m:1+t:2,m:1+t:23` | 沪市 A 股（主板 t:2 + 科创板 t:23） |
| `m:0+t:81+s:2048` | 北交所（可选追加） |

> 沪深全市场 A 股完整写法（推荐）：
> `fs=m:0+t:6,m:0+t:80,m:1+t:2,m:1+t:23`

### 返回字段（`fields`）含义
| 字段 | 含义 | 单位 |
|------|------|------|
| `f12` | 股票代码 | - |
| `f13` | 市场标识（0=深, 1=沪） | - |
| `f14` | 股票名称 | - |
| `f2`  | 最新价 | 元 |
| `f3`  | 涨跌幅 | % |
| **`f62`**  | **主力净流入净额** | 元 |
| **`f184`** | **主力净流入净占比** | % |
| `f66` | 超大单净流入净额 | 元 |
| `f69` | 超大单净流入净占比 | % |
| `f72` | 大单净流入净额 | 元 |
| `f75` | 大单净流入净占比 | % |
| `f78` | 中单净流入净额 | 元 |
| `f81` | 中单净流入净占比 | % |
| `f84` | 小单净流入净额 | 元 |
| `f87` | 小单净流入净占比 | % |

> 口径说明：主力 = 超大单 + 大单。即 `f62 ≈ f66 + f72`。

### 返回结构（JSON）
```json
{
  "rc": 0,
  "data": {
    "total": 5412,          // 全市场匹配到的股票总数
    "diff": [
      {
        "f12": "600519",
        "f13": 1,
        "f14": "贵州茅台",
        "f2": 1450.00,
        "f3": 1.23,
        "f62": 352000000.0,   // 主力净流入 3.52 亿元
        "f184": 8.31,         // 占比 8.31%
        "f66": 210000000.0,   // 超大单净流入
        "f72": 142000000.0,   // 大单净流入
        "f78": -50000000.0,   // 中单净流入
        "f84": -302000000.0   // 小单净流入
      }
      // ... 全市场每只股票一行
    ]
  }
}
```
- 数据在 `data.diff` 数组里，每个元素 = 一只股票的当日累计资金流快照。
- `data.total` = 全市场股票总数，可用于判断是否需要翻页。
- 字段为 `null` 表示该股当日无该项数据（如停牌）。

## 二、个股分钟级资金流曲线接口（配合用）

> 主用接口锁定 TOP 名单后，对单只信号股调此接口，看资金是「持续流入」还是「尾盘出逃」。

### 接口地址与示例
```bash
curl -s -H "Referer: https://quote.eastmoney.com" \
  "https://push2.eastmoney.com/api/qt/stock/fflow/kline/get?lmt=0&klt=1&secid=1.600519&fields1=f1,f2,f3,f7&fields2=f51,f52,f53,f54,f55,f56,f57,f58,f59,f60,f61,f62,f63,f64,f65"
```

### 参数
| 参数 | 取值 | 含义 |
|------|------|------|
| `secid` | `1.600519` | `市场.代码`，1=沪 0=深 |
| `klt` | `1` | K 线周期，1=1分钟（当日分时） |
| `lmt` | `0` | 0=返回全部 |

### 返回结构
```json
{ "data": { "name": "贵州茅台", "code": "600519",
    "klines": [
      "2026-06-05 09:31,15992330.0,0.0,-15864730.0,18917372.0,-2925042.0",
      "...",
      "2026-06-05 15:00,-113929464.0,-379342.0,114308817.0,-331703292.0,217773828.0"
    ] } }
```
- `klines` 每行 = 一分钟一条记录，逗号分隔。
- 字段顺序：`时间, 主力净流入, 小单净流入, 中单净流入, 大单净流入, 超大单净流入`
- 数值为**当日累计**净流入（元），不是单分钟增量。
- 一个完整交易日 = 240 行（09:31–15:00）。

## 三、实盘测试 Checklist

1. **联通性**：盘中单发主用接口，确认 `HTTP 200`，`data.diff` 非空，`data.total > 5000`。
2. **实时性**：盘中隔几分钟各调一次，确认同一只股票的 `f62` 数值在变化（累计在增长/回落）。
3. **字段核对**：如某字段全为 null，用浏览器 F12 打开 [data.eastmoney.com/zjlx](https://data.eastmoney.com/zjlx/)，看网页实际请求的 `fields` 顺序，对照修正（东财字段编号偶有调整）。
4. **口径验证**：抽查一只票，确认 `f62 ≈ f66 + f72`（主力=超大单+大单）。
5. **限流测试**：按 5~8 次/秒的节奏连续请求，观察是否出现连接重置（`HTTP 000` / `RemoteDisconnected`）。

---

## 四、限流注意事项（重要）

- 东财对**单 IP** 有频率限制，**无间隔暴力连发**（实测约 70 次）会触发 TCP 连接重置，封锁窗口 >50 秒。
- **正常按节奏调用不会触发**。建议：
  - 复用项目现有 `data_source.py` 的令牌桶限流器（`_eastmoney_limiter`），`max_per_sec` 控制在 5~8。
  - GitHub Actions 是海外 IP，更敏感，节奏再放慢一档、并发 ≤ 8。
- **最大优化点**：选股扫描用「主用接口」**1 次拿全市场**并排序，而不是逐只查 5000 次——请求量直接降几个数量级，几乎不会限流。

## 五、与现有 `data_source.py` 的关系

- 现有 `fetch_capital_flow(code)` 用的是 `stock/get` 的 f137/f140/f143，逐只查**单只快照**。
- 本文档主用接口 = 把同类快照**一次性拿全市场 + 排序**，适合做选股前置筛选。
- 两者数据口径一致（都是东财 Level-1 资金流），可互为补充。

> 免费/口径声明：以上均为东财官网前端公开 HTTP 接口，无需 API Key；返回为 Level-1 口径（按成交单笔金额拆分超大/大/中/小单），非 Level-2 主动买卖盘。
