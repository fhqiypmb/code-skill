# 市场环境过滤设计文档

> 创建日期：2026-05-28
> 目的：在钉钉推送的"轮次汇总"前，附加大盘环境提示，帮助人工判断是否买入。
> 设计原则：**不拦截信号扫描、不修改打分、不影响 ML**，只在汇总顶部加一段市场状态提示。

---

## 一、核心思路

> **股票照常扫，信号照常推，但是在每轮汇总最上方，加一段"市场环境警告"。**
> 我（人）看到警告/禁止后，自己决定要不要买。

- 健康时：不显示任何提示（保持汇总清爽）
- 跌破时：显示 ⚠️ 警告
- 持续跌破 ≥6 根 30min K：显示 🚫 禁止买入

---

## 二、监控对象

| 角色 | 指数 | 代码 | 作用 |
|---|---|---|---|
| **主指数** | 上证指数 | 000001 | 任一进入禁止状态 → 全市场禁止 |
| **主指数** | 深证成指 | 399001 | 任一进入禁止状态 → 全市场禁止 |
| **辅指数** | 创业板指 | 399006 | 仅做文字提示，不影响主决策 |
| **辅指数** | 科创 50 | 000688 | 仅做文字提示，不影响主决策 |
| **辅指数** | 北证 50 | 899050 | 仅做文字提示，不影响主决策 |

> 主辅区别：**只有上证 / 深证才能触发"禁止买入"**；创业 / 科创 / 北证只在文字里附带提示当前是否在 60 均线下方。

---

## 三、判断规则（30 分钟周期）

所有判断都基于 **30 分钟 K 线**，比较 **收盘价 vs MA60**（30 分钟周期 60 均线）。

### 状态分级

| 状态 | 触发条件 | 显示 |
|---|---|---|
| ✅ **健康** | 当前 30min 收盘价 ≥ MA60 | **不显示**任何提示 |
| ⚠️ **警告** | 当前 30min 收盘价 < MA60，且连续低于 MA60 的根数 < 6 | 显示警告文字 |
| 🚫 **禁止买入** | 连续 ≥ 6 根 30min 收盘价 < MA60 | 显示禁止文字 |

### 关键定义

**"连续 N 根低于 MA60"**：从最新一根 30min K 线往前数，连续多少根的 **收盘价** 都低于当根的 MA60。一旦中间出现一根收盘 ≥ MA60，计数清零。

**"刚下穿"**：等同于"连续低于 < 6 根"，即第 1 ~ 5 根状态。

**"行情好转"**：收盘价重新站上 MA60 → 自动不显示提示（不需要额外解禁逻辑）。

## 四、决策合并逻辑

```
对 5 个指数分别算出状态：
  上证 = ✅ / ⚠️ / 🚫
  深证 = ✅ / ⚠️ / 🚫
  创业 = ✅ / ⚠️ / 🚫
  科创 = ✅ / ⚠️ / 🚫
  北证 = ✅ / ⚠️ / 🚫

主决策（看上证 + 深证）：
  上证 = 🚫 OR 深证 = 🚫   →  全局禁止
  上证 = ⚠️ OR 深证 = ⚠️   →  全局警告
  上证 = ✅ AND 深证 = ✅   →  主指数健康

辅助提示（看创业 / 科创 / 北证）：
  只要任一辅指数 ≠ ✅，就在文字里附带说明
```

## 六、技术实现要点

### 1. 数据获取（**重要：现有函数不够用**）

**问题**：`data_source.fetch_index_kline()` 当前 URL 写死 `klt=101`（日线），拿不到 30 分钟数据。

**两个方案**：

**方案 a（推荐）**：新增 `fetch_index_kline_minute(index_code, period_minutes=30, count=80)`
- 东方财富的 URL 参数 `klt` 可选值：5=5min, 15=15min, 30=30min, 60=60min, 101=日线
- 复制现有 `fetch_index_kline` 的逻辑，把 `klt=101` 改成参数化即可
- 拉 80 根（约 10 个交易日的 30min K）足够算 MA60

**方案 b**：把 `fetch_index_kline` 加一个 `klt=101` 默认参数，向后兼容

建议方案 a，新增函数更清晰、不影响现有 RS 计算的稳定性。

### 2. 北证 50 数据源验证

北证 50（899050）东方财富的 secid 前缀是否为 0 或 1，要实测一下。如果拉不到数据，**第一版可以不上北证**，先做 4 个指数。

### 3. MA60 计算

```python
def calc_ma(closes: list[float], period: int = 60) -> float | None:
    if len(closes) < period:
        return None
    return sum(closes[-period:]) / period
```

注意：判断"当根"是否低于 MA60 时，MA60 应该用**截至当根的 60 根**算，不是用最新 MA60 去比较所有历史根。判断"连续 N 根"时，每根都要用各自截止位的 MA60。

### 4. 缓存与频率

- 30min K 线一根 30 分钟才更新一次，**一轮扫描内（约 5-10 分钟）拉一次足够**
- 建议在 `run_full_round` 开头一次性拉好 5 个指数，传入 `_format_round_summary`
- 拉取失败时降级：跳过该指数显示，不影响整轮推送

### 5. 文件组织

新增独立模块：`stocks/stock_monitor/market_env.py`

```
market_env.py
├── INDEX_CONFIG: 5 个指数的代码、名称、是否主指数
├── fetch_index_30min(code) -> list[K]    # 包装 data_source
├── calc_index_status(klines) -> dict     # 算单指数状态
│   返回 {'level': 'healthy'/'warning'/'forbidden',
│         'consec_below': int,
│         'close': float, 'ma60': float, 'diff_pct': float}
├── check_market_environment() -> dict    # 算 5 个指数 + 综合决策
│   返回 {'overall': 'healthy'/'warning'/'forbidden',
│         'indices': {'上证': {...}, ...},
│         'has_warning': bool}
└── format_market_warning(env) -> str     # 生成 markdown，健康时返回空串
```

### 6. 接入点

`monitor.py:_format_round_summary()` 开头加 3 行：

## 八、不做的事（明确边界）

为避免过度设计，第一版**不做**以下事情：

1. ❌ 不拦截扫描（信号正常扫、正常推、ML 正常记录）
2. ❌ 不动 V2 十条硬规则
3. ❌ 不做"自动降级阈值"（人看警告自己决定）
4. ❌ 不做加权打分（主就是主、辅就是辅，规则简单清晰）
5. ❌ 不做市场宽度 / 涨跌停家数 / 北向资金（数据源额外接入成本高，先做最核心的）
6. ❌ 不做"解禁等待 N 根确认"（站上 MA60 立即认为好转，符合"不显示就是好转"的简洁原则）

## 十、待确认事项

请确认以下设计点是否符合预期，再动手实现：

1. **辅指数是否包括北证 50**？北证流动性差、相关性弱，可能噪音大。是否先只做创业 + 科创？
2. **警告的"刚下穿"是否需要过滤毛刺**？例如收盘价低于 MA60 但差值 < 0.1% 时，是否仍算跌破？
3. **盘前 / 盘后**（非交易时段）跑监控时，30min K 还是上一交易日最后一根。是否需要在文字里说明"基于昨日收盘 30min K"？
4. **如果某个指数 K 线拉取失败**（网络异常），是显示"❓数据缺失"还是直接跳过该行？
5. **ML 埋点**这一步要不要和市场环境提示一起上线？还是先观察文字提示稳定后再埋？
