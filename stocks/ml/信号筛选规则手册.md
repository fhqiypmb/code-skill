## 二、V1 原规则（4条）

### 规则1：股价 ≥ 10 元

| 属性名 | 说明 |
|--------|------|
| `sc_close`（收盘价） | 信号发出当天的收盘价，即买入参考价 |

**单独效果**:

| | 全部1927条 | sc_close ≥ 10 (1470条) |
|---|:---:|:---:|
| 达标率 | 22.8% | 23.5% |
| 高弹性率 | 25.1% | 27.6% |
| 平均最大涨幅 | 5.6% | 5.9% |

**作用**:
- 排除低价股（<10元）的噪音信号。低价股虽然涨幅空间看似大，但信号质量差，容易受游资操纵，波动无序
- 10元是A股市场一个心理分界线，低于10元的股票往往基本面偏弱或处于下跌趋势
- 单独看提升不大（+0.7pp），但为后续条件奠定基础

### 规则2：主力净流入 ≥ 2000万

| 属性名 | 说明 |
|--------|------|
| `an_capital_main_net_in`（主力净流入/万） | 当日主力资金净流入金额，单位万元。主力=超大单+大单 |

**单独效果**:

| | 全部 | ≥2000万 (337条) |
|---|:---:|:---:|
| 达标率 | 22.8% | 25.5% |
| 高弹性率 | 25.1% | 33.8% |
| 平均最大涨幅 | 5.6% | 7.1% |

**作用**:
- 确认有真实资金在推动，不是无量空涨
- 主力净流入≥2000万是"资金认可"的最低门槛，低于这个值可能是散户零散买入
- 对高弹性率的提升（+8.7pp）远大于达标率（+2.7pp），说明资金流入更多是预测"爆发力"而非"确定性"

**⚠️ 关键发现：资金流入不是越多越好！**

| 主力净流入阈值 | 样本数 | 达标率 | 高弹性率 | 平均最大涨幅 |

| ≥1000万 | 114 | 34.2% | 46.5% | 9.2% |

| **≥2000万** | **83** | **33.7%** | **42.2%** | **8.9%** |

| ≥5000万 | 42 | 23.8% | 31.0% | 7.2% |
| ≥8000万 | 24 | 25.0% | 33.3% | 7.5% |

> ⚠️ ≥500万时效果最好（达标率36.1%），≥2000万开始下降，≥5000万大幅下降。超大资金流入可能是高位出货信号

### 规则3：主力流入强度 1~20

| 属性名 | 说明 |
|--------|------|
| `an_capital_flow_ratio`（主力流入强度%） | 主力净流入金额占成交额的比例，反映资金介入的"效率"而非绝对量 |

**单独效果**:

| | 全部 | ratio 1~20 (904条) |
|---|:---:|:---:|
| 达标率 | 22.8% | 23.8% |
| 高弹性率 | 25.1% | 27.3% |
| 平均最大涨幅 | 5.6% | 5.9% |

**作用**:
- 排除两种极端：ratio<1 说明主力几乎没参与，ratio>20 说明资金占比异常（可能是对倒或突击拉升）
- 1~20 是正常的主力参与区间，代表资金"温和而有力"地介入
- 单独看提升不大，但与资金净流入组合后效果显著

**数据补充**: ratio 区间进一步拆解（在V1其余条件基础上）:

| ratio区间 | 样本数 | 达标率 | 高弹性率 | 平均最大涨幅 |
|:---------:|:------:|:------:|:--------:|:------------:|
| **1~5** | **17** | **41.2%** | **58.8%** | **11.0%** |
| 1~10 | 63 | 36.5% | 42.9% | 9.8% |
| **1~20** | **83** | **33.7%** | **42.2%** | **8.9%** |
| 5~20 | 66 | 31.8% | 37.9% | 8.4% |

> 💡 ratio 1~5 是最佳区间（达标率41.2%，高弹性率58.8%），占比越低反而越好。高占比可能意味着"突击资金"，不持续

### 规则4：动能 ≥ 95

| 属性名 | 说明 |
|--------|------|
| `an_success_rate_dim_momentum`（动能评分） | 综合趋势得分和MACD强度的加权值，范围0~100。= trend_score × 0.7 + macd_strength × 0.3 |

**单独效果**:

| | 全部 | ≥95 (333条) |
|---|:---:|:---:|
| 达标率 | 22.8% | 28.2% |
| 高弹性率 | 25.1% | 41.4% |
| 平均最大涨幅 | 5.6% | 8.3% |

**作用**:
- **V1中区分度最高的单条条件**，高弹性率从25.1%直接拉到41.4%
- 动能高 = MACD金叉刚确认 + 均线多头排列 + 量价配合，代表趋势刚刚启动
- 95以上意味着MACD强度几乎满分（100），趋势正处于最强势阶段

**数据补充**: 动能阈值拆解（在V1前3条条件基础上）:

| 动能阈值 | 样本数 | 达标率 | 高弹性率 | 平均最大涨幅 |
|:--------:|:------:|:------:|:--------:|:------------:|
| ≥60 | 189 | 28.0% | 41.3% | 8.4% |
| ≥70 | 165 | 29.1% | 40.6% | 8.5% |
| ≥80 | 141 | 29.1% | 40.4% | 8.5% |
| ≥90 | 101 | 34.7% | 43.6% | 9.2% |
| **≥95** | **83** | **33.7%** | **42.2%** | **8.9%** |
| ≥100 | 51 | 37.3% | 45.1% | 9.4% |

> 💡 90和100是两个更好的切分点。95处于中间态。如果追求极致可收窄到100（达标率37.3%），如果追求样本量可放宽到90（达标率34.7%，多18条机会）

## 五、未达标股的失败原因分析

从11条未达标记录中，可以归纳出以下失败模式：

### 失败模式1：超高主力净流入（大资金出货）

| 股票 | 主力净流入 | 占比 | 大单净流入 | 实际收益 |
|------|:----------:|:----:|:----------:|:--------:|
| 比亚迪 | 19.5亿 | 13.13 | -1.15亿 | -2.1% |
| 飞龙股份 | 2.3亿 | 11.37 | -1706万 | +2.5% |
| 美好医疗 | 8846万 | 9.00 | -1133万 | -9.0% |

**特征**: 主力净流入极大但大单净流出 → 主力通过中小单制造"资金流入"假象，实际上大单在出逃

### 失败模式2：占比过高（突击拉升嫌疑）

| 股票 | 占比 | 实际收益 |
|------|:----:|:--------:|
| 澳弘电子 | 19.07% | -9.6% |
| 天成自控 | 17.97% | -1.7% |
| 狮头股份 | 18.42% | -1.5% |

**特征**: flow_ratio > 15，资金占比异常高，短期突击后缺乏持续性

### 失败模式3：高价股（资金效率低）

| 股票 | 价格 | 实际收益 |
|------|:----:|:--------:|
| 比亚迪 | 108.36 | -2.1% |

## 八、V2规则伪代码

```python
def v2_filter(record):
    """V2信号筛选规则"""

    # === V1 原规则 ===
    # 规则1: 股价 ≥ 10元
    if (record.get('sc_close') or 0) < 10:
        return False

    # 规则2: 主力净流入 ≥ 2000万
    if (record.get('an_capital_main_net_in') or 0) < 2000:
        return False

    # 规则3: 主力流入强度 1~20
    ratio = record.get('an_capital_flow_ratio') or 0
    if not (1 <= ratio <= 20):
        return False

    # 规则4: 动能 ≥ 95
    if (record.get('an_success_rate_dim_momentum') or 0) < 95:
        return False

    # === V2 新增规则 ===
    # 规则5: 当日涨跌幅 ≥ 3%
    if (record.get('an_quote_change_pct') or 0) < 3:
        return False

    # 规则6: 大单净流入 < 4000万
    if (record.get('an_capital_big_net_in') or 0) >= 4000:
        return False

    # 规则7: 周期 = 日线
    if record.get('period') != '日线':
        return False

    return True
```
