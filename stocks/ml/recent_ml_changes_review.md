# 最近 ML 模型改动梳理与后续修正建议

> 生成日期：2026-06-06  
> 范围：仅梳理近期 `stocks/ml` 机器学习模型逻辑相关改动，不包含股票列表和普通信号数据更新。

## 一、总体结论

最近几次 ML 模型改动的大方向是合理的：

1. 短线模型从“是否触达目标价”改为“持有 5 日净赚 >5%”，标签口径更统一、更贴近真实收益。
2. 潜力模型从“5 日内最大涨幅 >=8%”上调为“>=15%”，和涨幅排序模型拉开目标差异。
3. 涨幅模型继续保留“5 日内最大涨幅 >=8%”，定位为排序模型，目前是三类模型中实战效果最明确的一个。
4. 新增特征覆盖率护栏，自动剔除历史覆盖率过低的稀疏特征，能有效降低训练/推理分布偏移风险。
5. 新增 `ml_thresholds.json` 动态导出阈值，周报端已经开始使用动态阈值。

但当前模型验证仍有几个明显问题：

1. `train/test` 时间切分按样本数量切，导致同一天样本可能同时出现在训练集和测试集。
2. 报告中的 accuracy 对类别不平衡问题有误导性，不能证明模型有效。
3. 胜率/潜力模型虽然使用 isotonic 校准，但当前概率桶结果并不支持“输出概率≈真实概率”的强表述。
4. 实时推送 `monitor.py` 里涨幅模型图标阈值仍残留旧的 `67/60`，没有同步当前训练导出的 Top20/Top30 阈值。
5. 部分实时推送文案仍使用旧模型高命中率描述，和当前报告指标不一致。

---

## 二、各项改动评价

### 1. 短线胜率模型标签改为“5 日净赚 >5%”

当前逻辑：

- `SHORTLINE_PROFIT_THRESHOLD_PCT = 5.0`
- 标签为：`actual_return > 0.05`

评价：**合理**。

原来的 `reached_target` 是“5 日内最高价是否碰到目标价”。目标价本身由技术规则生成，每只股票目标价宽窄不同，容易变成“目标价设置是否宽松”的标签，不一定代表真实收益。

改成统一收益口径后：

- 标签可比性更强；
- 更贴近真实落袋收益；
- 不再依赖个股目标价宽窄；
- 方便和整体基准胜率比较。

当前样本统计：

| 指标 | 数量 | 比例 |
|---|---:|---:|
| 已标注样本 | 2770 | - |
| 5 日净赚 >5% | 484 | 17.47% |
| 旧 `reached_target` | 599 | 21.62% |

新标签更严格，也更贴近实战。

---

### 2. 潜力模型门槛从 8% 上调到 15%

当前逻辑：

- 潜力模型标签：`max_gain_pct >= 15%`
- 涨幅排序模型标签：`max_gain_pct >= 8%`

评价：**方向合理，但样本较少，需要谨慎使用**。

如果潜力模型继续使用 8%，会和涨幅排序模型高度重叠。上调到 15% 后，潜力模型专门识别“大涨弹性”，目标更清晰。

当前样本统计：

| 标签 | 数量 | 比例 |
|---|---:|---:|
| `max_gain_pct >= 8%` | 687 | 24.80% |
| `max_gain_pct >= 15%` | 283 | 10.22% |

15% 正样本更稀缺，符合“大涨潜力”定位。

当前报告中潜力模型高分桶：

| 概率区间 | 信号数 | 命中数 | 命中率 |
|---|---:|---:|---:|
| `>=60%` | 0 | 0 | 0.0% |
| `50%-60%` | 0 | 0 | 0.0% |
| `40%-50%` | 4 | 0 | 0.0% |
| `30%-40%` | 21 | 7 | 33.3% |
| `<30%` | 529 | 51 | 9.6% |

`30%-40%` 桶相对基准有明显提升，但样本只有 21 个，稳定性不足。

建议：潜力模型可以作为弹性辅助信号，不宜单独作为强买入依据。

---

### 3. 涨幅排序模型保留 8% 标签

当前逻辑：

- 标签：`max_gain_pct >= 8%`
- 模型：`RandomForestClassifier`
- 不做 isotonic 校准；
- 输出作为排序分数，而不是严格概率。

评价：**目前三类模型中最有实战参考价值**。

当前测试集 Top-N 效果：

| Top N | 样本数 | 命中率 | 相对基线提升 |
|---|---:|---:|---:|
| Top 50% | 277 | 34.3% | 1.4x |
| Top 30% | 166 | 42.2% | 1.8x |
| Top 20% | 110 | 47.3% | 2.0x |
| Top 10% | 55 | 47.3% | 2.0x |

整体 8% 涨幅基准约 24.8%，Top20 达到 47.3%，说明排序能力是存在的。

注意事项：

- Top10 与 Top20 命中率相同，说明顶部再细分的区分度一般；
- 输出值应理解为排序分数，不应解释成真实概率；
- 更适合作为“优先级排序/过滤条件”。

---

### 4. 特征覆盖率护栏

当前逻辑：

- `MIN_FEATURE_COVERAGE = 0.5`
- 候选特征中，非空覆盖率低于 50% 的自动剔除。

评价：**非常必要，且是近期最明确有效的改动之一**。

原因：近期新增的 `sc_mk_*` 市场环境字段在历史样本中覆盖率极低。当前已标注样本覆盖率约：

| 特征 | 覆盖率 |
|---|---:|
| `sc_mk_bz_below` | 1.08% |
| `sc_mk_cyb_below` | 1.08% |
| `sc_mk_kc_below` | 1.08% |
| `sc_mk_overall_code` | 1.08% |
| `sc_mk_sh_below` | 1.08% |
| `sc_mk_sz_below` | 1.08% |

如果这些字段参与训练，训练时几乎全是 0，但线上推理时可能有真实值，会造成明显分布偏移。

覆盖率护栏能避免：

> 训练时几乎恒为 0，推理时突然有值。

因此这个改动非常合理。

---

### 5. 动态阈值导出 `ml_thresholds.json`

当前阈值文件内容大意：

```json
{
  "win": {
    "baseline": 17.5,
    "high": 40.0
  },
  "potential": {
    "baseline": 10.2,
    "high": 30.0
  },
  "gain": {
    "baseline": 23.6,
    "top20": 43.3,
    "top30": 31.2
  }
}
```

评价：**机制合理，但高分线稳定性不足**。

优点：

- 比写死阈值更好；
- 周报可以随每次训练自动调整染色标准；
- 涨幅模型 Top20/Top30 阈值尤其应该动态更新。

问题：

胜率模型当前高分线 `>=40%` 来自测试集中 11 个样本、3 个命中，样本量太小。虽然相对基准有提升，但统计置信度不足。

建议：

- 阈值可以继续动态导出；
- 报告中应明确“高分线是历史测试分数档，不代表真实概率”；
- 可考虑提高 `min_samples`，或用滚动多期测试合并计算阈值。

## 三、当前主要问题

### 问题 1：时间切分存在同日混杂

当前报告显示：

```text
训练截止: 2026-05-18
测试起始: 2026-05-18
```

这说明同一天样本被拆进训练集和测试集。

风险：

同一天的信号共享大盘环境、行业轮动、资金状态和扫描背景。如果同一天样本同时在训练和测试中，测试效果可能偏乐观。

建议：按日期分组切分，保证某一天全部属于训练集或测试集。

示例思路：

```python
dates = sorted({r['date'] for r in labeled})
split_date = dates[int(len(dates) * 0.8)]
train_records = [r for r in labeled if r['date'] < split_date]
test_records = [r for r in labeled if r['date'] >= split_date]
```

更进一步可以做 walk-forward validation。

---

### 问题 2：Accuracy 指标误导性较强

短线胜率模型报告：

```text
训练集准确率: 88.76%
测试集准确率: 86.10%
```

但测试集短线正样本率约 13.36%。如果模型永远预测负类，准确率约 86.64%。

因此当前 86.10% accuracy 不能证明模型有效，甚至低于“全预测负类”的朴素基线。

更应该看：

- Precision@TopN；
- 高分桶命中率；
- Lift；
- 平均实际收益；
- 最大回撤；
- 分数单调性。

建议：报告中弱化 accuracy，把 Top-N / Lift / 收益回测作为核心指标。

---

### 问题 3：概率校准表述过强

报告当前表述类似：

> 胜率/潜力模型经 isotonic 校准，输出概率可当真实概率读。

但当前胜率概率桶并不支持这个说法：

| 概率区间 | 实际命中率 |
|---|---:|
| `>=40%` | 27.3% |
| `35%-40%` | 20.0% |
| `30%-35%` | 23.1% |
| `25%-30%` | 15.2% |

如果概率校准良好，`>=40%` 桶实际命中率应接近 40% 或更高。当前更像“有排序意义的分数”，不是严格概率。

建议：

- 文案改为“分数具有排序参考意义”；
- 不要说“概率≈真实概率”；
- 校准方式改为时间序列校准，例如训练集前段训练模型、训练集后段校准，最后测试集完全留出。

---

### 问题 4：实时推送涨幅阈值仍残留旧值

问题位置：`stocks/stock_monitor/monitor.py`

目前实时推送中涨幅图标仍使用旧阈值：

```python
gain_icon = '🔥' if ml_gain >= 67 else ('⭐' if ml_gain >= 60 else '💡')
```

出现位置：

- 单条非普通信号推送；
- 汇总推送行内展示。

但当前训练导出的阈值是：

- Top20：43.3
- Top30：31.2

因此实时推送会把很多当前模型认为 Top20/Top30 的信号误显示为低等级。

建议：

1. `monitor.py` 读取 `stocks/ml/ml_thresholds.json`；
2. 取 `gain.top20` 作为 🔥 阈值；
3. 取 `gain.top30` 作为 ⭐ 阈值；
4. 读取失败时再回退旧默认值 `67/60`。

示例思路：

```python
def _get_gain_cuts() -> tuple[float, float]:
    thresholds_file = os.path.join(PARENT_DIR, 'ml', 'ml_thresholds.json')
    try:
        with open(thresholds_file, 'r', encoding='utf-8') as f:
            thresholds = json.load(f)
        gain_thresholds = thresholds.get('gain', {}) if isinstance(thresholds, dict) else {}
        return (
            float(gain_thresholds.get('top20', 67) or 67),
            float(gain_thresholds.get('top30', 60) or 60),
        )
    except Exception:
        return 67.0, 60.0


def _get_gain_icon(ml_gain: float) -> str:
    gain_fire, gain_warm = _get_gain_cuts()
    if ml_gain >= gain_fire:
        return '🔥'
    if ml_gain >= gain_warm:
        return '⭐'
    return '💡'
```

然后替换两处：

```python
gain_icon = _get_gain_icon(ml_gain)
```

### 问题 5：实时推送胜率颜色注释过时

`monitor.py` 中 `_get_probability_color()` 的注释仍描述旧模型效果，例如：

```text
>=40%: 测试集实测精度~89%
>=35%: 测试集实测精度~52%
```

但当前报告里 `>=40%` 桶实际命中率是 27.3%，该注释已经明显过时。

建议：

- 删除“实测精度 xx%”这类硬编码说明；
- 改为读取 `ml_thresholds.json` 中的 `win.high` 和 `win.baseline`；
- 或至少将注释改成“不保证真实概率，仅用于颜色分档”。

## 四、建议后续修正优先级

### P0：修实时推送涨幅阈值

原因：这是明确 bug，当前线上展示和训练阈值不一致。

修法：`monitor.py` 读取 `ml_thresholds.json`，用 `gain.top20/top30` 替换硬编码 `67/60`。

---

### P1：按日期分组切分训练/测试集

原因：当前同日混杂会让测试结果偏乐观。

修法：不要按样本数量直接切，改为按日期切。

---

### P1：报告改用 Top-N / Lift 作为核心指标

原因：accuracy 在类别不平衡下误导性强。

建议新增：

- 短线胜率模型 Top50/30/20/10 命中率；
- 潜力模型 Top50/30/20/10 命中率；
- 各 Top 档平均 actual_return；
- 各 Top 档平均 max_gain_pct；
- 和随机基准的 lift。

---

### P2：弱化概率校准文案

原因：当前概率桶不支持“概率≈真实概率”。

建议文案从：

> 输出概率可当真实概率读。

改为：

> 输出分数经过校准处理，具备一定排序参考意义；实际可信度以测试集概率桶和 Top-N 命中率为准。

---

### P2：校准方式改为时间序列校准

当前 `CalibratedClassifierCV(cv=5)` 是普通交叉校准，不适合时间序列。建议后续改成：

1. 历史前 70%：训练基础模型；
2. 中间 10%：校准概率；
3. 最近 20%：最终测试；
4. 或使用 walk-forward validation。

---

## 五、推荐实战使用方式

当前更稳妥的使用方式是：

1. `ml_predict_gain` 作为主排序信号；
2. `ml_predict_potential` 作为弹性辅助信号；
3. `ml_predict_prob` 作为保守过滤信号；
4. 不要单独迷信 `ml_predict_prob` 的绝对概率数值；
5. 优先关注：涨幅 Top20 + 潜力高 + 规则匹配度高的交集。

简化策略：

```text
优先级 1：ML涨 >= Top20 阈值，且 ML潜 >= 高分线
优先级 2：ML涨 >= Top20 阈值，且规则匹配度高
优先级 3：ML潜高但胜率一般，降低仓位观察
避免：只因为 ML胜率数值高就重仓
```

---

## 六、最终判断

最近 ML 模型改动不是乱改，核心方向是对的：

- 标签更贴近收益；
- 模型目标更清晰；
- 涨幅排序模型有明显 lift；
- 稀疏特征护栏解决了一个真实分布偏移风险；
- 阈值动态化是正确方向。

但当前还不能把报告里的“准确率”和“概率”当成强实盘依据。更合理的定位是：

> 当前模型已经可以作为辅助排序和筛选工具，尤其是 `ml_predict_gain`；但概率解释、阈值稳定性和回测严谨性还需要进一步加强。
