# Auto Proxy - 自动代理端口检测与配置

## 技能描述
自动扫描并检测本地所有可用的代理端口，找到可用的HTTP代理并进行自动配置。支持智能识别和跳过SOCKS5端口，仅推荐可用的HTTP代理。

## 使用方法

### 自动检测代理
直接说"检测代理"、"查找代理"或"配置代理"，系统会自动运行检测脚本。

**示例：**
- "检测代理"
- "查找代理端口"
- "配置代理"
- "自动代理"

### 手动运行脚本
如果需要手动运行检测脚本，在终端中执行：
```powershell
powershell -ExecutionPolicy Bypass -File "C:\Users\10413\Desktop\test\skills\auto_proxy\autoProxy.ps1"
```

或者在技能目录下执行：
```powershell
.\autoProxy.ps1
```

## 功能特点
- ✅ 自动扫描本地所有监听端口
- ✅ 智能测试每个端口的代理功能
- ✅ 自动识别并跳过SOCKS5端口
- ✅ 推荐最佳可用的HTTP代理
- ✅ 支持临时配置和永久配置
- ✅ 验证代理连接并显示IP信息
- ✅ 彩色输出，易于阅读

## 检测流程
1. **扫描端口**：扫描所有本地监听的127.0.0.1端口
2. **测试代理**：测试每个端口是否为可用的HTTP代理
3. **结果汇总**：显示所有可用的HTTP代理和SOCKS5代理
4. **推荐配置**：推荐最佳代理并提供配置命令
5. **即时应用**：可选择立即应用代理配置到当前会话

## 支持的代理软件
- Clash / Clash for Windows
- V2rayN / V2ray
- Shadowsocks
- 其他支持HTTP代理的软件

## 注意事项
- 脚本需要在PowerShell环境中运行
- 代理软件需要已经启动并开启本地监听
- 临时配置仅在当前终端会话有效
- 永久配置需要管理员权限，重启终端后生效
- SOCKS5代理在PowerShell中不受支持，但可用于其他工具
