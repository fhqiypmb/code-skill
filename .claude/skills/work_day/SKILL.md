# Work Day - 工作日报生成器

> 指令：`生成日报` / `写日报`
> 产出：1条约150字工作内容 + 200字个人总结

---

## 1. 执行流程

```
读取 history.md → 去重检查 → 生成日报+总结 → 输出 → 追加历史 → 清理30天前记录
```

### 历史文件

- 路径：`D:\code-skill\.claude\skills\work_day\history.md`
- 每次生成**必须先读取**，生成后**必须追加写入**
- 写入格式：`- [YYYY-MM-DD] 领域: xxx | 关键词: xxx | 角度: xxx`
- 超过30天的记录在写入时自动清理

---

## 2. 去重规则

| 维度 | 不可重复窗口 | 降级条件 |
|------|-------------|---------|
| 技术领域 | 最近 5 天 | 全部用过则放宽为 3 天 |
| 标题关键词 | 最近 15 天 | — |
| 切入角度 | 最近 7 天 | — |

---

## 3. 写作风格

### 核心定位

**技术备忘录，不是工作汇报。** 只陈述技术机制的关系、约束和特征，不体现"做了什么"或"做到哪"。读完后无法判断是已做、在做还是打算做。

### 风格对比

| ❌ 禁止（有动作感） | ✅ 正确（纯陈述） |
|---------------------|-------------------|
| 对Layer层级的声明结构重新划分 | Layer层级的声明结构直接影响组件样式与全局样式的覆盖关系 |
| 调整Layer的引入时序 | Layer的引入时序决定跨模块样式的作用域边界 |
| 针对样式污染问题进行排查 | 跨模块样式污染的根源在于级联维度上的边界模糊 |

### 禁用词

| 类别 | 词汇 |
|------|------|
| 动作 | 进行、处理、解决、修复、调整、梳理、划分、整理、收拢、实现、搭建、接入、迁移、重构 |
| 完成 | 完成、搞定、做完 |
| 效率 | 效率、性能提升、优化效果、速度、耗时、显著提升、明显改善 |
| 学习 | 学习、研究、探索 |
| 排查 | 排查、发现、定位、存在、部分、某些、个别 |
| 业务 | 业务、项目、需求、产品 |
| 时态 | 今天、目前、当前、此前、之前、之后、接下来 |
| 数据 | 秒数、fps、百分比、MB 等具体数字 |
| 语气 | 了 |

### 其他约束

- 开头句式每次随机，禁止与上次相同
- 用词表述每次随机变化，避免固定搭配
- 纯文本，不含代码片段

## 4. 技术领域池

Vue生态、React生态、Three.js/WebGL、构建工具链、TypeScript类型系统、Node后端、CSS工程化、微前端、前端监控/埋点、Web Workers/多线程、PWA/Service Worker、GraphQL、状态管理方案、前端安全、自动化测试、低代码/可视化搭建、WebRTC/音视频、Canvas2D绑定

---

## 5. 标题规则

从技术领域中选取**具体知识点**展开，不直接用领域名称：

| ❌ 不要 | ✅ 应该 |
|--------|--------|
| Vue响应式优化 | 基于Proxy的依赖收集机制调优 |
| React状态管理 | useReducer配合Context的跨层级状态同步 |
| Three.js渲染优化 | 基于InstancedMesh的批量渲染策略 |
| Webpack配置 | SplitChunks分包策略与动态导入的配合 |
| 微前端 | qiankun沙箱环境下全局状态透传链路 |
| 前端监控 | 基于PerformanceObserver的长任务采集策略 |
