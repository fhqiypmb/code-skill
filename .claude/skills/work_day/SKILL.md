# Work Day - 工作日报与总结生成器

## 技能描述
自动生成前端开发相关的工作日报和总结，帮助快速记录和汇报工作内容。

## 使用方法

### 生成日报
直接说"生成日报"或"写日报"，系统会自动生成2条150字左右的前端开发工作内容。

**示例：**
- "生成日报"
- "写日报"
- "今天的日报"

### 生成总结
说"生成总结"或"写总结"，系统会基于之前的日报内容生成200字左右的个人思考和感悟。

**示例：**
- "生成总结"
- "写总结"
- "今天的总结"

## 功能特点
- ✅ 日报自动生成2条，每条约150字
- ✅ 内容涵盖Vue、React、Three.js、Vite、Webpack、TypeScript等前端技术
- ✅ 纯功能性描述，专注技术实现细节
- ✅ 总结包含个人思考和感悟
- ✅ 内容真实可信，符合实际工作场景

## 日报生成规则
- 不要直接使用框架名称作为标题（如"Vue组件优化"、"React Hooks封装"）
- 从技术框架中选取具体的知识点进行展开，聚焦某一个细分技术点
- 标题应体现具体的技术实现细节，而非泛泛的框架功能
- 两条日报内容应来自不同的技术领域，避免重复

### 标题示例
| ❌ 不要这样写 | ✅ 应该这样写 |
|--------------|--------------|
| Vue响应式优化 | 基于Proxy的依赖收集机制调优 |
| React状态管理 | useReducer配合Context实现跨层级状态同步 |
| Three.js渲染优化 | 基于InstancedMesh的批量渲染策略调整 |
| Webpack配置 | SplitChunks分包策略与动态导入配合优化 |
| TypeScript类型 | 条件类型与infer关键字在泛型工具中的应用 |

## 总结生成规则
- 总结为200字左右的连贯段落，不分段
- 将两条日报内容关联在一起进行整体思考
- 包含个人感悟和技术思考，体现工作价值

## 交互流程
- 生成日报后，提示用户：`是否生成总结？(y/n)`
- 用户输入 `y` 或 `Y` 时生成总结
- 用户输入 `n` 或 `N` 时结束

## 注意事项
- 日报内容为纯文本形式，不包含示例代码
- 避免使用"学习"等字眼，突出工作成果
- 总结需要基于当天日报内容进行关联思考
- 不要出现具体数字化指标（如秒数、fps、百分比、MB等）
- 描述效果使用"显著提升"、"明显改善"等定性表述
- 避免使用"完成"、"完成了"等结果性字眼，使用"处理"、"进行"、"优化"等进行时表述
