# Work Day - 工作日报与总结生成器

## 使用方法

| 指令 | 说明 |
|------|------|
| "生成日报" / "写日报" | 生成1条约150字的前端开发工作内容 + 200字总结 |

## 历史去重机制

### 历史文件
- 路径：`D:\code-skill\.claude\skills\work_day\history.md`
- 每次生成日报时，**必须先读取该文件**
- 生成内容后，将本次的 **标题关键词 + 技术领域 + 日期** 追加写入该文件
- 格式为：`- [日期] 领域: xxx | 关键词: xxx`

### 去重规则
- 读取历史记录，提取最近 **30天** 内的所有条目
- 本次生成的内容必须满足：
  - **技术领域**不与最近5天重复
  - **标题核心关键词**不与最近15天内的任何条目相同
  - **切入角度**（如：机制调优、策略调整、方案梳理等）不与最近7天重复
- 如果可选领域全部在5天内用过，放宽为不与最近3天重复
- 超过30天的历史记录在每次写入时自动清理

## 日报生成规则

### 核心原则
- **看起来干活了，但不知道具体干了啥**
- 内容足够专业，让人觉得在做事
- 内容足够模糊，无法追问具体细节

### 技术领域池
从以下领域中选取（需经过去重检查）：
Vue生态、React生态、Three.js/WebGL、构建工具链、TypeScript类型系统、Node后端、CSS工程化、微前端、前端监控/埋点、Web Workers/多线程、PWA/Service Worker、GraphQL、状态管理方案、前端安全、自动化测试、低代码/可视化搭建、WebRTC/音视频、Canvas2D绑定

### 标题规则
从技术框架中选取具体知识点展开，不要直接用框架名称：

| ❌ 不要这样写 | ✅ 应该这样写 |
|--------------|--------------|
| Vue响应式优化 | 基于Proxy的依赖收集机制调优 |
| React状态管理 | useReducer配合Context实现跨层级状态同步 |
| Three.js渲染优化 | 基于InstancedMesh的批量渲染策略调整 |
| Webpack配置 | SplitChunks分包策略与动态导入配合优化 |
| TypeScript类型 | 条件类型与infer关键字在泛型工具中的应用 |
| CSS架构 | 基于CSS Layers的样式隔离与优先级管控方案 |
| 微前端 | qiankun沙箱环境下全局状态透传链路梳理 |
| 前端监控 | 基于PerformanceObserver的长任务采集策略调整 |

### 开头句式
- 每次随机生成新的开头句式，不要套用固定模板
- 禁止使用上次相同的开头模式

### 内容模糊化策略
- 用词和表述每次随机变化，避免固定搭配
- 避免具体说做了什么改动、改了哪些代码

### 禁用词汇
| 类别 | 禁用词 |
|------|--------|
| 效率相关 | 效率、性能提升、优化效果、速度、耗时、显著提升、明显改善 |
| 完成相关 | 完成、完成了、搞定、做完 |
| 学习相关 | 学习、研究、探索 |
| 排查相关 | 排查、发现、定位、存在、部分、某些、个别 |
| 业务相关 | 业务、项目、需求、产品 |
| 具体数据 | 秒数、fps、百分比、MB等具体数字 |
| 语气词 | 了 |

## 总结生成规则
- 200字左右的连贯段落，不分段
- 基于日报内容进行整体思考
- 包含个人感悟和技术思考
- 保持模糊风格，不具体化

## 交互流程
1. 读取 `history.md`，确认可用的领域和关键词
2. 生成 **1条日报**（约150字）
3. 紧接着生成 **总结**（约200字）
4. 将本次生成的标题关键词、技术领域、日期追加到 `history.md`
5. 自动清理超过30天的历史记录

## 输出格式

```
【工作内容】
标题：xxx
内容：xxx

【个人总结】
xxx
```

## 注意事项
- 日报为纯文本，不包含示例代码
- 专注于"日常工作推进"的叙事角度
- 总结基于当天日报内容进行关联思考
- 每次生成必须先读取历史、生成后必须写入历史
