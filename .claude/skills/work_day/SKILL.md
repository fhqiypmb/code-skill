# 📝 Work Day - 工作日报生成器

> 指令：`生成日报` / `写日报` / `日报`
> 产出：1条技术笔记（约150字）+ 个人总结（约200字）

---

## 技术领域（7个）

```
Vue生态 | React生态 | ThreeJS | WebGL | 工程化 | TS | NodeJS
```

---

## 写作规范

### 内容要求

> **聚焦一点，不及其余**
> 
> 从7个领域中选一个，围绕该领域的**一个很小很具体的点**展开
> 
> 同一个点可以从不同角度写（比如：机制、原理、边界、坑、对比）

### 举例

| 领域 | 可写的内容 |
|:---|:---|
| Vue生态 | computed的缓存机制、watch的deep选项、v-for的key原理、setup的执行时机、provide/inject的响应式... |
| React生态 | useState的闭包问题、useEffect的清理时机、useMemo的依赖比较、Context的穿透问题、React18的批处理... |
| ThreeJS | BufferGeometry的内存、Raycaster的性能、Object3D的层级、材质的复用、requestAnimationFrame的调度... |
| WebGL | 着色器的精度、纹理mipmap、Framebuffer切换、attribute传值、blend模式... |
| 工程化 | Vite的预构建、SplitChunks策略、tree-shaking条件、依赖扁平化、符号链接... |
| TS | infer逆推类型、extends分发、keyof推导、satisfies收窄、never排他... |
| NodeJS | EventLoop顺序、Buffer内存分配、Stream背压、模块缓存、nextTick优先级... |

---

## 去重规则

- **1年内不重复**：过去365天内写过的技术点不能再写
- **记录方式**：在SKILL.md末尾记录已使用过的技术点（用日期+领域+角度标识）
- **去重逻辑**：生成前检查历史，确保不重复

---

## 写作风格

### 核心定位

> **技术备忘录，不是工作汇报**
> 
> 只陈述技术机制本身，不体现「做了什么」

### 禁用词

| 类别 | 词汇 |
|:---|:---|
| 动作 | 进行、处理、解决、修复、调整、梳理、划分、整理、收拢、实现、搭建、接入、迁移、重构 |
| 完成 | 完成、搞定、做完 |
| 效率 | 效率、性能提升、优化、速度、耗时、提升、改善 |
| 学习 | 学习、研究、探索 |
| 时态 | 今天、目前、当前、此前、之前、之后、接下来 |
| 语气 | 了 |
| 业务 | 业务、项目、需求、产品 |

### 格式要求

- 纯文本，不含代码
- 开头随机变化（浅析/漫谈/探究/解析/剖析 + 技术点）
- 用词每次换说法

---

## 输出格式

```
【技术笔记】
标题：xxx（具体到某一个小点）

内容：xxx（约150字，只讲这一个点）

【个人总结】
xxx（约200字，连贯段落，个人感悟+技术思考）
```

---

## 执行步骤

1. 读取SKILL.md末尾的history数据
2. 选一个最近没写过的技术点（领域+角度组合）
3. 按格式生成内容
4. 输出结果
5. 将本次记录追加到SKILL.md末尾
