# 技能系统学习助手 (Skill Helper)

## 📚 什么是技能（Skill）？

技能是 DeepV Code AI 助手的扩展能力模块。通过技能系统，您可以：
- 为 AI 助手添加专业领域知识
- 提供可执行的脚本工具
- 定制化 AI 的行为和响应方式

## 🎯 技能的类型

### 1. 知识型技能（无脚本）
这类技能为 AI 提供特定领域的知识和指导，不包含可执行脚本。

**适用场景：**
- 专业领域指导（如编程规范、设计模式）
- 工作流程说明
- 最佳实践建议

### 2. 脚本型技能（有脚本）📜
这类技能包含可执行的脚本文件，可以执行具体的任务。

**适用场景：**
- 文档处理（PDF、Excel、PPT）
- 数据转换和分析
- 自动化任务

## 🛠️ 如何创建新技能

### 基本结构

每个技能需要放在 `.deepvcode/skills/技能名称/` 目录下，至少包含一个 `SKILL.md` 文件。

```
.deepvcode/
└── skills/
    └── 你的技能名称/
        ├── SKILL.md          # 必需：技能定义文件
        ├── script.py         # 可选：Python 脚本
        ├── script.js         # 可选：JavaScript 脚本
        └── README.md         # 可选：详细文档
```

### SKILL.md 模板

```markdown
---
name: 你的技能名称
description: "简短描述这个技能的功能"
---

# 技能标题

## 功能说明
详细描述这个技能能做什么。

## 使用方法
说明如何使用这个技能。

## 示例
提供具体的使用示例。

## 注意事项
任何需要注意的地方。
```

## 📖 使用技能的完整流程

### 步骤 1: 列出所有可用技能
```
请帮我列出所有可用的技能
```

AI 会调用 `list_available_skills` 工具显示所有技能列表。

### 步骤 2: 查看技能详情
```
请帮我查看 [技能名称] 的详细信息
```

AI 会调用 `get_skill_details` 工具显示该技能的完整文档。

### 步骤 3: 激活并使用技能
```
请使用 [技能名称] 技能
```

AI 会调用 `use_skill` 工具激活技能。如果是脚本型技能，会显示脚本的使用说明。

### 步骤 4: 执行脚本（仅脚本型技能）
按照技能文档中的说明，使用 `run_shell_command` 执行脚本。

## 💡 创建技能的最佳实践

### 1. 清晰的命名
- 使用小写字母和下划线
- 名称要能体现技能的用途
- 例如：`code_review`, `data_analyzer`, `api_tester`

### 2. 完整的文档
- 在 SKILL.md 中详细说明功能
- 提供清晰的使用示例
- 列出所有参数和选项

### 3. 合理的范围
- 一个技能专注于一个特定领域
- 不要让单个技能过于复杂
- 相关功能可以创建多个技能

### 4. 错误处理
- 在脚本中添加适当的错误处理
- 提供有用的错误信息
- 考虑边界情况

## 🎓 实用技能创建示例

### 示例 1: 代码审查助手

```markdown
---
name: code_reviewer
description: "帮助审查代码质量、发现潜在问题并提供改进建议"
---

# 代码审查助手

## 功能
- 检查代码风格和规范
- 识别潜在的 bug
- 提供性能优化建议
- 检查安全问题

## 使用方法
激活此技能后，AI 将以专业代码审查者的身份：
1. 分析您的代码
2. 指出问题和改进点
3. 提供具体的修改建议
```

### 示例 2: API 文档生成器（脚本型）

```markdown
---
name: api_doc_generator
description: "从代码自动生成 API 文档"
has_scripts: true
---

# API 文档生成器

## 功能
从代码注释和函数签名自动生成 API 文档。

## 脚本使用
激活技能后，使用以下命令：

\`\`\`bash
python .deepvcode/skills/api_doc_generator/generate.py --input src/ --output docs/api.md
\`\`\`

## 参数说明
- `--input`: 源代码目录
- `--output`: 输出文档路径
- `--format`: 文档格式（markdown/html）
```

## 🔍 调试技能

如果技能无法正常工作：

1. **检查文件路径**
   - 确保 SKILL.md 在正确的目录
   - 路径使用正确的分隔符（Windows: `\`）

2. **验证 YAML 格式**
   - SKILL.md 开头的 YAML front matter 格式正确
   - name 和 description 字段存在

3. **测试脚本**
   - 单独测试脚本是否能运行
   - 检查依赖是否已安装

4. **查看日志**
   - 注意 AI 返回的错误信息
   - 根据错误信息调整

## 📚 推荐学习路径

1. **入门阶段**
   - 先使用现有技能，熟悉工作流程
   - 理解知识型和脚本型技能的区别

2. **实践阶段**
   - 创建简单的知识型技能
   - 尝试添加自己的专业领域知识

3. **进阶阶段**
   - 创建包含脚本的技能
   - 整合外部工具和 API

4. **高级阶段**
   - 创建复杂的工作流技能
   - 分享您的技能给其他用户
